Arch Linux - General installation procedure

  1. Introduction
  2. Pre-installation
  3. Installation

Introduction

This guide shows how to install Arch Linux in UEFI mode. It will be assumed that a USB installer is used. The instructions here are heavily based on ArchWiki's installation guide. When some of the instructions here do not work, check ArchWiki.

Pre-installation

Power on the computer and boot from the installation USB.

Set keyboard layout

$ loadkeys <map>

where <map> is one of the available layouts from

$ ls /usr/share/kbd/keymaps/**/*.map.gz

omitting path and file extension.

Connect to the Internet

Verify that there is a working Internet connection

$ ping archlinux.org

If no connection is available, stop the dhcpcd service with

$ systemctl stop dhcpcd@<interface>

where <interface> can be tab-completed. Then configure the network as described in ArchWiki's Network configuration.

Update the system clock

$ timedatectl set-ntp true

Partition the disks

The following partition scheme will be used for installing Arch Linux in UEFI mode:

Mount pointTypeComments
/boot/EFIEFI partition
/ext4root partition
/homeext4home partition
swapswapin a non solid-state drive

Execute 

$ fdisk /dev/sd<D>

where <D> is the devide identifier of the device to partition.

Create a new partition table

Creating a new GUID Partition Table (GPT) on a disk with data on it will erase all the data on the disk.

If a new GPT is desired, on fdisk's prompt type g to create it.

Create the new partitions

To create a new partition, on fdisk's prompt:

  • Type n;
  • Specify the partition number;
  • Specify the first sector. It is advisable to use the default value;
  • Specify the last sector. It is advisable to specify it by using +<size>[K|M|G|T].

To list the partitions, on fdisk's prompt type p.

To change the partition type, on fdisk's prompt:

  • Type t;
  • Specify the partition number;
  • Specify the partition type. Type L to list all possible values.

Write changes to disk

To write the changes to disk, on fdisk's prompt type w.

Format the partitions

Format the partitions by executing

$ mkfs.fat -F32 /dev/sd<De>
$ mkfs.ext4 /dev/sd<Dr>
$ mkfs.ext4 /dev/sd<Dh>

where <De>, <Dr> and <Dh> refer to the device identifier and partition number of the EFI, root and home partitions, respectively. The value D is different for different devices, so change it according to the device corresponding to each partition.

If a swap partition was created, initialise it with mkswap

$ mkswap /dev/sd<Ds>
$ swapon /dev/sd<Ds>

where <Ds> refers to the device identifier and partition number of the swap partition.

Mount the file systems

Mount the root partition under /mnt, and the EFI and home partitions under the root partition

$ mount /dev/sd<Dr> /mnt
$ mkdir -p /mnt/boot/efi
$ mount /dev/sd<De> /mnt/boot/efi
$ mkdir /mnt/home
$ mount /dev/sd<Dh> /mnt/home

where <De>, <Dr> and <Dh> are as explained above.

Installation

Select the mirrors

Packages to be installed must be downloaded from mirror servers, which are defined in /etc/pacman.d/mirrorlist. The higher a mirror is placed in this file, the more priority it is given when downloading a package. Edit the file to assign the desired priority (e.g. prefer mirrors geographically closer.) This file will be copied to the new system.

Install initial packages

$ pacstrap /mnt base linux linux-firmware e2fsprogs man-db man-pages openssh unzip vim wget networkmanager firewalld base-devel git

Generate /etc/fstab file

$ genfstab -U /mnt >> /mnt/etc/fstab

Change root into the new system

$ arch-chroot /mnt

Time zone

$ ln -sf /usr/share/zoneinfo/<Region>/<City> /etc/localtime

where <Region> is one of the available options from

$ ls /usr/share/zoneinfo

and <City> is one of the available options from

$ ls /usr/share/zoneinfo/<Region>

Then, generate /etc/adjtime

$ hwclock --systohc

Locale

Open /etc/locale.gen, uncomment the desired localisation(s), and generate them with

$ locale-gen

Create /etc/locale.conf and add lines setting the variables LANG and LANGUAGE to the desired values

/etc/locale.conf
LANG=<localisation>
LANGUAGE=<list_languages>

Make keyboard layout persistent by creating /etc/vconsole.conf and adding a line setting the variable KEYMAP to the desired layout (see Set keyboard layout above)

/etc/vconsole.conf
KEYMAP=<map>

Network configuration

Create /etc/hostname and add a line with the desired hostname <hostname>

/etc/hostname
<hostname>

Create /etc/hosts and add matching entries

/etc/hosts
127.0.0.1    localhost
127.0.1.1    <hostname>.localdomain    <hostname>
::1          localhost

If the system has a permanent IP address, it should be used instead of 127.0.1.1.

Complete the network configuration for the newly installed environment as described in ArchWiki's Network configuration. Finally, verify that there is a working Internet connection

$ ping archlinux.org

Set root password

$ passwd

Create unprivileged user

Create user <user>

$ useradd -U -u <uid> -m -k /dev/null <user>
$ passwd <user>

Open /etc/sudoers using visudo

$ EDITOR=vim visudo

and configure privileges for <user> by uncommenting Defaults targetpw and adding permissions

/etc/sudoers
...
Defaults targetpw
...
<user> ALL=(ALL) ALL

Boot loader

Install a capable boot loader (see the special installation steps).

Reboot

Exit the chroot environment by executing exit or pressing Ctrl+D.

Optionally, unmount all the partitions

$ umount -R /mnt

Restart the machine by executing reboot.
